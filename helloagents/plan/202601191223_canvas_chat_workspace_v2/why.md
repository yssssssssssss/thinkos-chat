# 变更提案: 双栏工作台（自由画布/图层 + 对话）联动

## 需求背景
- 目前主入口仅渲染 `src/chat/ChatView`（`App.tsx` 中 `Canvas` 被注释），缺少“对话产物 → 画布整理/二次处理”的连续工作流。
- 目标结构参考 https://www.lovart.ai/：页面左侧为画布，右侧为对话；对话/工具产生的媒体自动沉淀到画布；在画布上选中媒体后出现对应工具栏。
- 前提约束：完整保留当前项目中的全部能力（Chat 多模型/会话/Prompt/工具/编辑能力 + 现有 `Canvas.tsx` 工作流节点能力等），在此基础上做结构调整与联动增强。

## 已确认需求（关键结论）
1. 左侧画布：更像 lovart 的**自由排版/图层式画布**（非节点流图）。
2. 画布资产：**跨会话汇总**，固定不随会话切换。
3. 持久化：需要保存画布上的**位置/缩放**等布局，刷新后仍存在（包含视口平移/缩放）。
4. 自动同步范围：
   - ✅ 包含：Video2Gif / Png2Apng / Particleize 等工具产物（自动同步到画布）
   - ❌ 不包含：用户上传、参考图（不自动同步到画布）
5. 工具栏：V1 使用指定按钮集合，同时必须保留**可扩展机制**（便于后续迭代添加动作）。

## ⚠️ 待确认（开工前最后 2 个问题）
1. 对话中 AI 生成的图片/编辑后的图片（assistant `imageResponses`）是否也自动同步到画布？
   - 建议默认：✅ 同步（仅采集 assistant 的 `imageResponses`，避免把用户上传/参考图带入）
2. 画布持久化的容量策略（IndexedDB 有配额）：超限时希望怎么处理？
   - [A] 停止自动同步并提示（最安全）
   - [B] 自动清理最旧的画布资源（更省心）
   - [C] 让用户选择清理策略（更可控）

## 变更内容
1. **结构调整**：新增 Workspace 双栏布局（左画布/右对话），支持拖拽分割线调整宽度（可选：折叠/全屏）。
2. **自由画布（图层式）**：新增自由画布模块（媒体卡片自由排版），支持选中、拖拽、缩放/平移视口（V1 不强制旋转/分组/对齐线）。
3. **媒体自动沉淀**：将允许范围内的产物自动加入画布，并在画布中可视化管理（跨会话汇总）。
4. **选中工具栏**：选中图片/GIF/视频后展示对应工具栏；工具栏动作采用可扩展注册机制。
5. **能力保留**：不移除/降级现有 ChatView 与 `Canvas.tsx` 的任何能力入口；在 Workspace 中保留访问路径（建议：左侧顶部提供“画板/工作流”切换）。

## V1 不做清单（避免范围失控）
- 不实现高级图层能力：分组/对齐辅助线/历史回退/多人协作/云端同步。
- 不强制与会话做双向绑定（例如点击画布资产跳转到对应消息），后续可迭代。
- 不自动同步用户上传文件与参考图（严格按“已确认需求”）。

## 影响范围
- **新增模块**
  - `src/workspace/*`：Workspace 布局 + 跨面板联动上下文
  - `src/board/*`：自由画布（图层式）+ 资产模型 + 工具栏扩展点 + 持久化存储
- **改动模块**
  - `App.tsx`：入口切换为 Workspace
  - `src/chat/ChatView.tsx`：容器化尺寸适配 + 产物上报（不改核心业务逻辑）
  - `src/chat/components/modals/*`：工具产物生成后自动上报到画布
  - `Canvas.tsx`：容器化尺寸适配；保留为“工作流”子模式入口

## 成功标准（验收）
- **结构**：页面左画布右对话，分割条可拖拽；窄屏可折叠画布保证聊天可用。
- **自动同步**：Video2Gif / Png2Apng / Particleize 产物生成后自动出现在画布。
- **画布交互**：资产可选中/拖拽排版；画布可平移/缩放；刷新后布局与视口保持。
- **工具栏**：选中图片/GIF/视频显示工具栏；默认按钮可用；新增按钮无需重写画布核心代码（有扩展点）。
- **能力不丢失**：ChatView 现有能力可用；`Canvas.tsx` 工作流能力可从 Workspace 访问。

## 核心场景

### 场景 1：默认进入工作台
- 进入页面后左侧展示自由画布，右侧展示对话。
- 拖拽分割条调整左右宽度（最小宽度保护）。

### 场景 2：工具产物自动沉淀到画布
- 在右侧使用 Video2Gif / Png2Apng / Particleize 生成产物后，左侧画布自动新增对应媒体卡片（可预览）。
- 同一产物不会重复添加（去重策略见 how.md）。

### 场景 3：画布自由排版 + 持久化
- 用户在画布上拖拽调整卡片位置；滚轮缩放画布视口；刷新页面后布局与视口保持。

### 场景 4：选中工具栏（可扩展）
- 选中图片/GIF：展示按钮（设为参考图/下载/复制/移除）。
- 选中视频：展示按钮（下载/移除/转 GIF）。
- 后续可通过“动作注册”扩展更多按钮（无需侵入画布核心）。

### 场景 5：保留工作流画布能力
- 从 Workspace 左侧进入“工作流/节点画布”（复用现有 `Canvas.tsx`），确保原有能力仍可用。

## 风险评估
- **实现复杂度**：自由画布需要坐标系、视口变换、选中/拖拽/缩放等交互处理。
- **持久化与存储配额**：要刷新后仍存在，工具产物 Blob 需写入 IndexedDB；需容量策略（上限/清理/提示）。
- **布局回归风险**：`ChatView` 与 `Canvas.tsx` 当前均使用 `w-screen h-screen`，容器化可能引发滚动/弹窗定位回归。
- **内存管理**：ObjectURL 需在移除/卸载/清理时 `URL.revokeObjectURL`，避免泄漏。
