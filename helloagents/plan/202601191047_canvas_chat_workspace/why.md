# 变更提案: 双栏工作台（画布 + 对话）联动

## 需求背景
- 目前主入口仅渲染 `src/chat/ChatView`（`App.tsx` 中 `Canvas` 被注释），画布与对话是“互斥存在”，缺少“对话产物 → 画布整理/二次处理”的工作流。
- 目标是调整为类似 https://www.lovart.ai/ 的结构：页面左侧为画布，右侧为对话；对话产出的媒体自动沉淀到画布；在画布上选中媒体后出现对应工具栏。
- 前提约束：完整保留当前项目中的全部能力（Chat 的多模型/会话/Prompt/工具/编辑能力 + Canvas 工作流节点能力等），在此基础上做结构调整与联动。

## 变更内容
1. **结构调整**：新增“工作台”布局，左侧画布、右侧对话，并支持拖拽分割线调整宽度（可选：折叠/全屏）。
2. **媒体同步**：对话模式中生成的图片 / GIF / 视频（包含内置工具产物）自动同步到左侧画布成为可选资产。
3. **选中工具栏**：在画布上选中图片 / GIF / 视频后，展示对应工具栏（V1 聚焦：下载/复制/设为参考图/删除/打开相关工具）。
4. **能力保留**：不移除/降级现有 ChatView 与 Canvas 的任何功能入口；仅做布局承载与跨面板联动增强。

## 影响范围
- **模块**
  - `App`：入口渲染由单一 Chat 改为 Workspace 双栏布局
  - `Chat`：容器化尺寸适配 + 媒体产出事件上报/同步
  - `Canvas`：容器化尺寸适配 + 新增媒体资产节点 + 工具栏
  - `Storage`：必要时新增“画布资产/布局”存储（优先先内存态，按需升级到 IndexedDB）
- **关键文件（预估）**
  - `App.tsx`
  - `Canvas.tsx`
  - `types.ts`
  - `src/chat/ChatView.tsx`
  - `src/chat/components/modals/Png2ApngModal.tsx`
  - `src/chat/components/modals/Video2GifModal.tsx`
  - `src/chat/components/particleize/*`
  - 新增：`src/workspace/*`、`components/nodes/MediaNode.tsx` 等

## 核心场景

### 需求 1：双栏结构
#### 场景：默认进入工作台
- 进入页面后，左侧显示画布，右侧显示对话；二者高度跟随窗口，互不撑破。
- 允许拖拽分割线调整左右宽度（最小宽度保护）。

#### 场景：小屏/窄屏可用
- 在窄屏下可折叠左侧画布（或改为上下布局），保证对话主流程可用。

### 需求 2：媒体同步
#### 场景：对话生成图片
- 当对话产生图片结果（包含多模型并行结果、图像编辑产物）后，画布自动出现对应资产节点/卡片。

#### 场景：对话生成 GIF / APNG
- 当对话或工具产生 GIF/APNG 后，画布自动出现对应资产节点，并可预览与下载。

#### 场景：对话/工具生成视频
- 当工具产出视频（如 webm）后，画布自动出现视频节点，支持播放预览与下载（V1）。

### 需求 3：选中工具栏
#### 场景：选中图片 / GIF
- 工具栏（V1）至少包含：设为参考图（同步到 Chat 输入区）、下载、复制链接/复制到剪贴板、从画布移除、打开图像编辑。

#### 场景：选中视频
- 工具栏（V1）至少包含：下载、从画布移除、转 GIF（打开 Video2Gif 工具或二次处理入口）。

## 风险评估
- **布局回归风险**：`ChatView`/`Canvas` 当前均使用 `w-screen h-screen`，改为容器化可能影响内部滚动与弹窗定位。
- **同步去重与一致性**：同一消息可能多次触发状态更新，需确保资产只添加一次，且切换会话时资产正确切换。
- **Blob/ObjectURL 内存管理**：工具产出多为 Blob + ObjectURL，需在移除/切换时释放，避免内存泄漏。
- **需求不确定项**：画布是否“按会话隔离”、资产布局是否需要持久化、工具栏的必选功能集合仍需确认（见执行前确认问题）。
